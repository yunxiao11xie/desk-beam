## 1. 项目概述

### 1.1 背景

Korvo-2-LCD 板卡拥有 800×480 RGB 彩色 LCD、电容触摸屏、双麦克风阵列、音频编解码器、WS2812 RGB LED 等丰富的多媒体外设，但目前仅运行了一个 GIF 播放器 DEMO，大量硬件资源闲置。

与此同时，我们日常在 PC 上使用网易云/QQ 音乐等播放器听歌时，缺少一个能实时显示歌词、营造氛围的实体桌面伴侣。

### 1.2 目标

打造一个**桌面音乐伴侣**设备：

- **不播放音频** — 音频由 PC 播放，避免板子音质不如 PC 的问题
- **不浪费算力** — 板子专注 UI 渲染、氛围灯控制、语音交互
- **充分利用硬件** — LCD、触摸、麦克风、RGB LED、按键全部用上
- **无线连接** — PC 与板子通过 WiFi WebSocket 实时通信

### 1.3 核心理念

```
PC 端         →   管"数据源 + 控制执行"  (SMTC 监听 + 歌词 API + 播放控制)
ESP32 板端    →   管"显示 + 交互 + 氛围" (歌词 UI + 触摸/按键/语音 + RGB 灯)
```

## 2. 硬件资源清单

### 2.1 板卡型号

**ESP32-S3-Korvo-2-LCD**，搭载 **ESP32-S31** 芯片（RISC-V 双核 320MHz）。

### 2.2 资源分配

| 硬件 | 在本项目中的用途 | 是否新启用 |

| **ADC 按键** SET/MODE/VOL-/VOL+ (GPIO42) | 快捷控制：播放/暂停、模式切换、上下曲 | ✅ 已用，新用途 |
| **板载麦克风 ×2** → ES7210 ADC | 语音命令词拾音（ESP-SR 唤醒词识别） | 🆕 **新启用** |

### 2.3 引脚复用检查

| 外设 | 占用引脚 | 与本项目其他外设冲突？ |
|------|---------|---------------------|
| LCD RGB | GPIO 8~19, 33~36, 40, 43~45 | ❌ 无冲突 |
| 触摸 I2C | GPIO 0 (SDA), GPIO 1 (SCL) | ❌ 无冲突 |
| WS2812 | GPIO 37 | ❌ 无冲突 |
| ADC 按键 | GPIO 42 | ❌ 无冲突 |
| ES7210 麦克风 | GPIO 9~10, 16~17 | ⚠️ GPIO 9 与 LCD_DATA1 共用 — 需确认原理图 |
| ES8311 音频 | GPIO 16~18 | ⚠️ GPIO 16 与 LCD_DATA9 共用 — 需确认原理图 |
| SD 卡 | GPIO 4, 7, 15 | ❌ 无冲突 |
| Camera (备用) | GPIO 3, 11~14, 21, 38~42 | ⚠️ GPIO 42 与 ADC 按键共用 |

> ⚠️ **需要确认**：ES7210/ES8311 与 LCD 之间是否存在引脚复用冲突。如果共用引脚通过板级跳线/复用器切换，则需在软件中管理外设独占访问。

## 4. PC 端设计

### 4.1 技术栈

| 组件 | 选型 | 理由 |
|------|------|------|
| **SMTC 访问** | `winrt` (Python) | 直接调用 Windows Runtime API，无需第三方播放器 SDK |
| **歌词 API** | `lrclib.net` | 免费、无需认证、返回同步 LRC 歌词 |
| **WebSocket** | `websockets` (asyncio) | 原生 async、轻量、社区活跃 |
| **HTTP** | `requests` | 简单可靠的同步 HTTP 库 |

> 已有基础实现：`pc_tools/windows_media_server.py`

### 4.2 工作流程

```
启动 → 请求 SMTC 会话管理器 → 创建 WebSocket Server
  │
  ├─ 协程1: poll_media_loop (每秒)
  │    ├─ 获取当前媒体会话
  │    ├─ 读取 MediaProperties (title/artist/album)
  │    ├─ 读取 Timeline (duration/position)
  │    ├─ 读取 PlaybackInfo (state/shuffle/repeat)
  │    ├─ 检测 "换歌" → 查歌词 → 广播 song_info + lyrics
  │    ├─ 检测 "进度跳转" → 广播 song_info
  │    └─ 检测 "状态变化" → 广播 song_info
  │
  ├─ 协程2: position_loop (每秒)
  │    └─ 广播 position (type + position_ms + duration_ms + state)
  │
  └─ 协程3: WebSocket server
       ├─ 新客户端 → 立即推送当前 song_info + lyrics
       └─ 接收消息 → handle_command(play_pause/next/prev/...)
```

### 4.3 命令行接口

```bash
# 基本用法
python pc_tools/windows_media_server.py

# 自定义端口
python pc_tools/windows_media_server.py --port 9000

# 局域网内指定地址
python pc_tools/windows_media_server.py --host 0.0.0.0
```

### 4.4 依赖安装

```bash
pip install winrt websockets requests
```

## 5. ESP32 端设计

### 5.1 技术栈

| 组件 | 选型 | 说明 |
|------|------|------|
| **框架** | ESP-IDF v5.x | 官方主推，组件化 |
| **GUI** | LVGL 9.x | 已在用，支持 PPA 硬件加速 |
| **WebSocket** | `esp_websocket_client` | ESP-IDF 官方组件 |
| **JSON** | `cJSON` | ESP-IDF 内置 |
| **语音** | `esp-sr` | 乐鑫语音识别组件（需验证 S31 兼容性） |
| **WiFi** | `esp_wifi` + `esp_netif` | 标准 WiFi Station 模式 |
| **SD 卡** | `sdmmc` + `fatfs` | 标准 SPI/MMC 驱动 |

### 5.2 模块划分

#### 5.2.1 通信模块 — `app_media_client`

```
app_media_client
├── 职责: 管理 WiFi + WebSocket 连接，接收/分发消息
├── 状态机:
│     DISCONNECTED → CONNECTING → CONNECTED → (断线重连)
├── 接收处理:
│     song_info → 更新全局当前歌曲状态
│     lyrics    → 更新歌词缓存，通知 UI
│     position  → 更新播放进度
└── 发送处理:
       command  → 将用户操作打包为 JSON 发送到 PC
```

**断线重连策略：**
- 首次连接：WiFi 就绪后主动连接
- 断线检测：WebSocket `onClose` 回调
- 重连间隔：1s → 2s → 4s → 8s → 16s（指数退避，最大 30s）
- 重连时自动请求当前状态（PC 端会推送最新 song_info + lyrics）

> 详见第 7 章 UI 设计

#### 5.2.3 氛围灯模块 — `app_led_effects`

#### 5.2.4 语音模块 — `app_voice_cmd`

#### 5.2.5 按键模块

## 6. 通信协议

### 6.1 传输层

| 项目 | 值 |
|------|-----|
| 协议 | WebSocket (ws://) |
| 端口 | 8765 |
| 编码 | UTF-8 |
| 格式 | JSON |
| 心跳 | 应用层无需额外心跳（TCP keepalive 即可） |

### 6.2 消息格式

#### PC → ESP32

**song_info** — 歌曲信息（换歌/状态变化时发送）

```json
{
  "type": "song_info",
  "title": "晴天",
  "artist": "周杰伦",
  "album": "叶惠美",
  "duration_ms": 263000,
  "position_ms": 45000,
  "state": "playing",
  "shuffle": false,
  "repeat_mode": "off"
}
```

**lyrics** — 歌词数据（换歌时发送）

```json
{
  "type": "lyrics",
  "lines": [
    { "time_ms": 12000, "text": "故事的小黄花" },
    { "time_ms": 16000, "text": "从出生那年就飘着" },
    { "time_ms": 20000, "text": "童年的荡秋千" },
    { "time_ms": 24000, "text": "随记忆一直晃到现在" }
  ]
}
```

**position** — 播放进度（每秒广播）

```json
{
  "type": "position",
  "position_ms": 47000,
  "duration_ms": 263000,
  "state": "playing"
}
```

#### ESP32 → PC

**command** — 控制命令

```json
{
  "type": "command",
  "action": "play_pause"
}
```

支持的 `action` 值：

| action | 效果 | 备注 |
|--------|------|------|
| `play_pause` | 播放/暂停切换 | 最常用 |
| `next` | 下一曲 | |
| `prev` | 上一曲 | |
| `seek` | 跳转到指定位置 | 扩展：需传 `position_ms` 字段 |
| `set_volume` | 调整音量 | 扩展：需传 `volume` 字段 (0~100) |

### 7.3 歌词滚动算法

```
1. 收到 lyrics 消息 → 存入歌词行数组 lines[]
2. 收到 position 消息 → 更新当前播放进度 current_ms
3. 每 100ms LVGL 定时器:
   a. 二分查找 current_ms 在 lines[] 中的位置 idx
      → lines[idx].time_ms <= current_ms < lines[idx+1].time_ms
   b. idx 变化时 → 触发高亮行切换动画
   c. 计算当前行内进度:
      progress = (current_ms - lines[idx].time_ms)
               / (lines[idx+1].time_ms - lines[idx].time_ms)
   d. 可选: 根据 progress 微调高亮行的颜色透明度/缩放
4. 渲染:
   - 从 idx-2 到 idx+3 共 6 行可见
   - idx 行: 白色大字，left 指示符
   - idx-1 行: 浅灰，稍小
   - idx-2 及更早: 深灰，更小
   - idx+1 行: 浅灰
   - idx+2 及以上: 深灰
```

### 7.4 显示模式

用户可通过 MODE 键或触摸切换以下模式：

| 模式 | 显示内容 | 氛围灯联动 |
|------|----------|-----------|
| **Lyrics** (默认) | 滚动歌词 + 进度控制 | 歌词行切换脉冲 |
| **Now Playing** | 大封面图 + 歌名 + 进度条 | 封面主色调呼吸 |
| **Visualizer** | 实时波动频谱（扩展） | BPM 同步节奏 |
| **Info** | 播放列表/设备信息/系统状态 | 柔和渐变 |

## 8. RGB 氛围灯方案

### 8.1 硬件

- LED 型号: WS2812 (单颗，GPIO37)
- 驱动方式: RMT (已在用)
- 物理位置: 板载指示灯

> 扩展：可通过板载 WS2812 引脚外接 LED 灯带（如 30 颗/m 桌面氛围灯带），软件架构不变，只需修改 LED 数量参数。

### 8.2 效果模式

#### 模式 1: 歌词流动脉冲 (默认)

```
触发时机: 每切换一句歌词
效果:
  当前颜色 → 目标颜色渐变 (300ms)
  亮度脉冲: 40% → 100% → 60% (200ms 起振 + 400ms 衰减)
  颜色: 从歌词主色采样（取当前高亮行的平均色调）
  频率: 与歌词切换频率一致
```

#### 模式 2: 呼吸律动

```
触发时机: 持续运行
效果:
  颜色: 从歌曲 mood 推断（快歌=暖色，慢歌=冷色）
  周期: 与歌曲 BPM 同步（需扩展计算，默认 2s）
  亮度: 正弦波呼吸 30% ↔ 80%
```

#### 模式 3: 封面颜色同步

```
触发时机: 歌曲切换
效果:
  分析封面图主色调（由 PC 端采样并随 song_info 发送）
  颜色: 封面主色
  过渡: 4s 渐变到新色
```

#### 模式 4: 待机彩虹

```
触发时机: 暂停/无播放
效果:
  HSV 色相循环 (0° → 360°, 周期 8s)
  亮度: 40%
```

### 8.3 LED 状态机

```
       ┌────────────────────┐
       │                    │
       ▼                    │
  ┌─────────┐    ┌──────────┴──┐
  │ IDLE    │───►│ LYRIC_PULSE │

  └──────────┘    └────────────┘
```

## 9. 语音命令（待验证）

### 9.1 ESP-SR 组件

| 项目 | 内容 |
|------|------|
| 组件 | `esp-sr` (ESP Speech Recognition) |
| 模型 | `MultiNet` (中文命令词识别) |
| 唤醒词 | `wake word` (可选) |
| 输入 | 双麦克风 → ES7210 ADC → I2S → ESP-SR |

### 9.2 命令词集

| 命令词 | 触发动作 |
|--------|----------|
| "小乐小乐" | 唤醒词（可选） |
| "播放/暂停" | 播放/暂停切换 |
| "下一首" | 切到下一曲 |
| "上一首" | 切到上一曲 |
| "换灯" | 切换氛围灯模式 |
| "语音关闭" | 退出语音监听模式 |

### 9.3 兼容性风险

ESP32-S31 采用 RISC-V 架构，ESP-SR 组件文档目前标注支持 **ESP32 / ESP32-S2 / ESP32-S3**（均为 Xtensa 架构）。

| 可能性 | 说明 |
|--------|------|
| ✅ 最佳情况 | ESP-SR 已适配 RISC-V，`idf.py menuconfig` 中正常开启 |
| ⚠️ 中间情况 | ESP-SR 部分可用（如 MultiNet 可用，但 WakeNet 不可用） |
| ❌ 最差情况 | ESP-SR 暂未适配 RISC-V，语音功能需等乐鑫更新 |

**应对策略：**
1. **Phase 5 验证** — 在项目后期（Phase 5）实际测试 ESP-SR 在 S31 上的兼容性
2. **备用方案** — 如果 ESP-SR 不可用，语音功能降级为**按键 + 触摸**控制，项目其余部分不受影响
3. **后续升级** — 等乐鑫官方更新 ESP-SR 对 RISC-V 的支持后，再集成语音功能
